1 A Régua da Eficiência: Big-O

A Notação Big-O é a linguagem que usamos para descrever a performance. O objetivo de todo bom algoritmo é manter a curva de crescimento o mais plana possível conforme os dados (N) aumentam.

Curvas de Complexidade

Comparação do tempo de execução conforme N cresce.

O(1) - Constante

O tempo não muda, não importa se você tem 10 ou 1 milhão de itens. Ex: Acessar índice de Array, Chaves de Hash.

O(log n) - Logarítmico

Corta o problema pela metade a cada passo. Ex: Busca Binária, Árvores Balanceadas.

O(n) - Linear

Percorre item por item. O tempo cresce igual aos dados. Ex: Loop simples, Busca em Lista Ligada.

Hash Tables (O Mapa do Tesouro)

A única estrutura que oferece acesso Instantâneo O(1). Ela usa uma função matemática (Hash) para transformar uma chave (como um nome) diretamente em um endereço de memória.

  • ✓ Busca: O(1) médio.
  • ✓ Inserção: O(1) médio.
  • ! Colisões: Resolvidas com Listas Ligadas.
